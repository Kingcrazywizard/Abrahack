
# ENRUTAMIENTO DE TRAFICO CON IPTABLES + REDSOCKS

Podemos usar iptables + redsocks para enrutar TODO el tráfico de un sistema a través de un proxy, como por ejemplo nuestro proxy en AWS-EC2

---

### 1. Instalar redsocks
```bash
sudo apt update
sudo apt install redsocks
```

### 2. Crear archivo de configuración

Crea o edita el archivo `/etc/redsocks.conf`:
```bash
sudo nano /etc/redsocks.conf
```

Pega esto (ajústalo si tu proxy escucha en otro puerto):

```ini
base {
  log_debug = off;
  log_info = on;
  daemon = on;
  redirector = iptables;
}

redsocks {
  local_ip = 127.0.0.1;
  local_port = 14444;
  ip = 127.0.0.1;
  port = 8443;
  type = http-connect;
  login = "";
  password = "";
}
```

Guarda con `Ctrl+O` y cierra con `Ctrl+X`.

---

### 3. Levantar redsocks
```bash
sudo systemctl restart redsocks
sudo systemctl enable redsocks
```

Confirma que esté escuchando en el puerto 14444:
```bash
sudo ss -tulpn | grep 14444
```

---

### 4. Redireccionar tráfico con iptables

Este paso captura TODO el tráfico saliente y lo envía al proxy.

Primero crea una nueva cadena:
```bash
sudo iptables -t nat -N REDSOCKS
```

Luego redirige todo el tráfico TCP que no sea local ni de Tor:
```bash
# Evita que se proxee el tráfico local y el de tor
sudo iptables -t nat -A REDSOCKS -d 0.0.0.0/8 -j RETURN
sudo iptables -t nat -A REDSOCKS -d 10.0.0.0/8 -j RETURN
sudo iptables -t nat -A REDSOCKS -d 127.0.0.0/8 -j RETURN
sudo iptables -t nat -A REDSOCKS -d 169.254.0.0/16 -j RETURN
sudo iptables -t nat -A REDSOCKS -d 172.16.0.0/12 -j RETURN
sudo iptables -t nat -A REDSOCKS -d 192.168.0.0/16 -j RETURN
sudo iptables -t nat -A REDSOCKS -d 224.0.0.0/4 -j RETURN
sudo iptables -t nat -A REDSOCKS -d 240.0.0.0/4 -j RETURN

# Redirige todo el tráfico TCP al puerto de redsocks
sudo iptables -t nat -A REDSOCKS -p tcp -j REDIRECT --to-ports 14444

# Aplica esa cadena a la salida
sudo iptables -t nat -A OUTPUT -p tcp -j REDSOCKS
```

### 5. Verifica que se aplicaron 

```bash
sudo iptables -t nat -L -n -v
```

---

### 5. Probar que todo está saliendo por el proxy

Desde otro terminal
```bash
curl https://icanhazip.com
```
Un servicio más permisivo que `ifconfig.me` para testear tu IP externa vía Tor
Deberías ver una IP que no es la de tu servidor, sino la que sale a través de Tor o el proxy configurado en Privoxy.

---

### 6. (Opcional pero recomendado): Persistir reglas iptables

Para que se mantengan tras reinicio:
```bash
sudo apt install iptables-persistent
sudo netfilter-persistent save
```

---

### ¿Cómo deshacer los cambios?

Si querés volver a la normalidad:
```bash
sudo iptables -t nat -F OUTPUT
sudo iptables -t nat -F REDSOCKS
sudo iptables -t nat -X REDSOCKS
```
