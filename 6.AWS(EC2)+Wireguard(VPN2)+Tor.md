# AWS(EC2) + WIREGUARD VPN2 + TOR
 
## Información general

### ¿Por qué usar WireGuard junto a Privoxy/Tor?

    WireGuard aporta:
    - Cifrado de tráfico completo del sistema (no solo HTTP)
    - Acceso remoto privado a la instancia EC2
    - Interacción directa con herramientas backend 
    - IP privada estática en túnel virtual

    Privoxy + Tor aporta:
    - Navegación anónima a través de la red Tor
    - Ideal para tráfico selectivo o anonimato extremo  

Combinados, ofrecen seguridad, anonimato y control.

### Requisitos
    - Instancia (EC2) + Iptables & Redsocks
    - Privoxy + Tor
    - Puertos abiertos en Security Group (por ejemplo, `53000/UDP` para WireGuard)

### Rango de IPs privadas (túnel)
    Usaremos: `10.200.200.1/24`  
    Servidor: `10.200.200.1`  
    Cliente: `10.200.200.2`

### Puerto personalizado
    Usaremos `53000/UDP`

## 1. Generación de llaves con `wg`

```bash
wg genkey | tee wg1_server_private.key | wg pubkey > wg1_server_public.key
wg genkey | tee wg1_client_private.key | wg pubkey > wg1_client_public.key
```

## Crea archivo /etc/wireguard/wg1.conf

```bash
[Interface]
Address = 10.200.200.1/24
ListenPort = 53000
PrivateKey = wg1_server_private.key

[Peer]
PublicKey = wg1_client_public.key
AllowedIPs = 10.200.200.2/32
PersistentKeepalive = 25
```

## 2. Configura reglas iptables VPN2(wg1)

Queremos que solo el tráfico entrante a través del túnel VPN wg1, se redirija a Redsocks (y por tanto a Tor), sin afectar:
    - Tráfico local
    - Tráfico proveniente de wg0 (VPN limpia)
    - Servicios internos 

Entonces:

```bash
## Permitir puerto UDP de WireGuard (53000)
sudo iptables -A INPUT -p udp --dport 53000 -j ACCEPT
## Permitir reenvío desde wg1 hacia Internet
sudo iptables -A FORWARD -i wg1 -o enX0 -j ACCEPT
## Permite el tráfico de respuesta volver desde Internet (eth0) hacia tu cliente VPN (wg1).
sudo iptables -A FORWARD -i enX0 -o wg1 -m state --state RELATED,ESTABLISHED -j ACCEPT
## Enmascara (NAT) para trafico saliente
sudo iptables -t nat -A POSTROUTING -s 10.200.200.0/24 -o enX0 -j MASQUERADE
```

Si solo deseas que el proxy Tor (Privoxy) sea accesible desde conexiones a través de VPN (wg1).

```bash
##  Permite tráfico entrante a 8443 (Privoxy), únicamente si ese tráfico llega a través de la interfaz wg1
sudo iptables -A INPUT -i wg1 -p tcp --dport 8443 -j ACCEPT
## Permite el trafico de respuesta
sudo iptables -A FORWARD -i enX0 -o wg1 -m state --state RELATED,ESTABLISHED -j ACCEPT
```

Para que realmente funcione debes anular la regla creada en AWS(EC2)+Iptables&Redsocks
`sudo iptables -t nat -I OUTPUT -p tcp --dport 8443 -j RETURN`

```bash
#Identifica el número de regla
sudo iptables -t nat -L OUTPUT -n --line-numbers
##Eliminala
sudo iptables -t nat -D OUTPUT <line_number>
```

Verifica reglas cargadas
```bash
sudo iptables -t nat -L -n -v --line-numbers
```
Guardalas
```bash
sudo netfilter-persistent save
```

## 3. Levanta el servicio

```bash
sudo wg-quick up wg1
sudo systemctl enable wg-quick@wg1
```

Verifica que corre

```bash
sudo systemctl status wg-quick@wg1 ## Puede aparecer inactivo si aun no hay trafico
sudo wg show
```

Cierra la conexión por 8443 en tu SG si lo prefieres

## 4. Configuración en Android (WireGuard app)

```bash
[Interface]
PrivateKey = (client_private.key)
Address = 10.200.200.2/32
DNS = 1.1.1.1

[Peer]
PublicKey = server_public.key
Endpoint = IP_PÚBLICA(EC2):53000
AllowedIPs = 0.0.0.0/0, ::/0
PersistentKeepalive = 25
```

## 5. Interacción con Proxy 

- **Desde el VPN podés acceder a tu proxy privado**: por ejemplo, `http://10.100.100.1:8443`
- Ideal para enrutar ciertas herramientas por Tor (via proxy) con `wg1` y otras por salida directa `wg0`
- Desde Android: configura navegador o app para usar el proxy en `10.100.100.1:8443`
