# AWS(EC2) + WIREGUARD VPN 

## Información general

### ¿Por qué usar WireGuard?

    WireGuard aporta:
    - Cifrado de tráfico completo del sistema (no solo HTTP)
    - Acceso remoto privado a la instancia EC2
    - Interacción directa con herramientas backend 
    - IP privada estática en túnel virtual

### Requisitos
    - Instancia (EC2) + Iptables (Redsocks)
    - Puertos abiertos en Security Group (por ejemplo, `42069/UDP` para WireGuard)

### Rango de IPs privadas (túnel)
    Usaremos: `10.100.100.1/24`  
    Servidor: `10.100.100.1`  
    Cliente: `10.100.100.2`

### Puerto personalizado
    Usaremos `42069/UDP`

## 1. Generación de llaves con `wg`

Primero, instalá WireGuard:

```bash
sudo apt update
sudo apt install wireguard -y
```

Generá las llaves (se almacenan localmente):

```bash
umask 077
wg genkey | tee server_private.key | wg pubkey > server_public.key
wg genkey | tee client_private.key | wg pubkey > client_public.key
##Usa cat para copiarlas y configurarlas con mayor facilidad
```

WireGuard usa criptografía moderna: **Curve25519 para intercambio de claves, ChaCha20 para cifrado**.

---

## 2. Activar IP forwarding

Edita el archivo de sysctl:

```bash
sudo nano /etc/sysctl.conf
```

Descomenta o añade:

```bash
net.ipv4.ip_forward=1
```

Aplica cambios:

```bash
sudo sysctl -p
```

---

## 3. Configuración del servidor WireGuard

Edita el archivo:

```bash
sudo nano /etc/wireguard/wg0.conf
```

```bash
[Interface]
Address = 10.100.100.1/24
ListenPort = 42069
PrivateKey = server_private.key

[Peer]
PublicKey = client_public.key
AllowedIPs = 10.100.100.2/32
PersistentKeepalive = 25
```

Verificar existencia y permisos del archivo:

```bash
sudo ls -l /etc/wireguard/wg0.conf
sudo chmod 600 /etc/wireguard/wg0.conf
```

---

## 4. Configuración de reglas iptables VPN(wg0)

```bash
## Evita que el tráfico de wg0 pase por Redsocks
sudo iptables -t nat -I OUTPUT -d 10.100.100.0/24 -j RETURN
## Permitir puerto UDP de WireGuard (42069)
sudo iptables -A INPUT -p udp --dport 42069 -j ACCEPT
## Permitir reenvío desde wg0 hacia Internet
sudo iptables -A FORWARD -i wg0 -o enX0 -j ACCEPT
## Permite el tráfico de respuesta volver desde Internet (eth0) hacia tu cliente VPN (wg0). 
sudo iptables -A FORWARD -i eth0 -o wg0 -m state --state RELATED,ESTABLISHED -j ACCEPT
## Enmascara (NAT) para trafico saliente
sudo iptables -t nat -A POSTROUTING -s 10.100.100.0/24 -o enX0 -j MASQUERADE
```

Guardalas
```bash
sudo netfilter-persistent save
```

---

## 5. Levantá el servicio:

```bash
sudo systemctl start wg-quick@wg0
sudo systemctl enable wg-quick@wg0
```

Verifica que corre:
```bash
sudo systemctl status wg-quick@wg0
sudo wg show
```

## ERRORES EN LA INSTALACIÓN

<details>
<summary>Limpieza total de configuración VPN (Reset WireGuard)</summary>

Detener el túnel y deshabilitar el servicio
```bash
sudo wg-quick down wg0
sudo systemctl disable wg-quick@wg0
```

Desinstalar WireGuard
```bash
sudo apt remove --purge wireguard wireguard-tools -y
```

Eliminar archivos de configuración
```bash
sudo rm -rf /etc/wireguard
sudo rm -f ~/server_private.key ~/server_public.key ~/client_private.key ~/client_public.key
```

Eliminar completamente archivos residuales de WireGuard
```bash
sudo wg-quick down wg0 2>/dev/null
sudo apt remove --purge wireguard wireguard-tools -y
sudo rm -rf /etc/wireguard
```

Eliminar o desactivar el servicio fallido
```bash
sudo systemctl disable wg-quick@wg0.service
sudo systemctl reset-failed wg-quick@wg0.service
```

Verifica que no exista más el servicio o procesos
```bash
sudo wg show
sudo ps aux | grep wg ##Probablemente aparezca uno debido a la busqueda
sudo systemctl list-units --failed
```

Elimina iptables activas
```bash
sudo iptables -D FORWARD -i wg0 -j ACCEPT
sudo iptables -t nat -D POSTROUTING -o eth0 -j MASQUERADE
sudo iptables -D INPUT -p udp --dport 42069 -j ACCEPT
```
</details>

## 5. Configuración en Android (WireGuard app)

1. Instala [WireGuard desde F-Droid o Play Store](https://play.google.com/store/apps/details?id=com.wireguard.android)
2. Toca "+" > "Crear desde cero"

```bash
[Interface]
PrivateKey = (client_private.key)
Address = 10.100.100.2/32
DNS = 1.1.1.1

[Peer]
PublicKey = server_public.key
Endpoint = IP_PÚBLICA(EC2):42069
AllowedIPs = 0.0.0.0/0, ::/0
PersistentKeepalive = 25
```

3. Activa el túnel

Verificá conexión con:

```bash
curl https://ifconfig.me
```

## 5. Conexión desde PC (Linux/Windows)

### Linux:
```bash
sudo nano /etc/wireguard/wg0.conf
```
misma configuración `.conf` de Android. Luego:

```bash
sudo systemctl start wg-quick@wg0
```

### Windows:
1. Instalá [WireGuard para Windows](https://www.wireguard.com/install/)
2. Creá un túnel con la misma configuración `.conf` que Android
3. Activá el túnel con un clic
