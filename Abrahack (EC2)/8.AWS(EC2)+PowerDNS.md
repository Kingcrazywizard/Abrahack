
## PowerDNS

- PowerDNS será utilizado como un servidor autoritativo para dominios personalizados y potencialmente como servidor de zona para TLDs (Top-Level Domains) privados como `.abrahack` en wt0

- operará sobre puertos personalizados para evitar conflictos y se integrará directamente con DNSCrypt para ofrecer un sistema completo de resolución seguro y cifrado.


## Arquitectura

| Red VPN | Motor DNS | TLD Personalizados | Reenvío Externo | Privacidad |
|---------|-----------|--------------------|------------------|------------|
| wg0     | PowerDNS  | Sí                 | DNSCrypt         | Alta       |
| wg1     | DNSCrypt  | No                 | Directo          | Máxima     |

## Instalación de PowerDNS y SQlite

```bash
sudo apt update
sudo apt install pdns-server pdns-backend-sqlite3 sqlite3 -y

```


## Crea manualmente el archivo de la base de datos SQLite:

```bash
sudo mkdir -p /etc/powerdns/db
```

Crear la base de datos SQLite y esquema
```bash
sudo sqlite3 /etc/powerdns/db/pdns.sqlite3 < /usr/share/pdns-backend-sqlite3/schema/schema.sqlite3.sql
```

Verificación de la Base de Datos (Opcional):

```bash
sudo sqlite3 /etc/powerdns/db/pdns.sqlite3
#  En el prompt de sqlite> :
#  sqlite> .tables
#  sqlite> .quit
```

Asegurá los permisos correctos
```bash
sudo chown -R pdns:pdns /etc/powerdns/db
sudo chmod 600 /etc/powerdns/db/pdns.sqlite3
```
Verificar los permisos del directorio /etc/powerdns/db

sudo chown pdns:pdns /etc/powerdns/db
sudo chmod 755 /etc/powerdns/db
ls -ld /etc/powerdns/db


## Editar el archivo de configuración:

```bash
sudo nano /etc/powerdns/pdns.conf
```

Agrega o modifica las siguientes líneas:

```ini
launch=gsqlite3
gsqlite3-database=/etc/powerdns/db/pdns.sqlite3

local-address=0.0.0.0
local-port=53053

#  Opcional: Habilitar servidor web (¡Asegurar en producción!)
#  webserver=yes
#  webserver-address=127.0.0.1
#  webserver-port=8082

#  Opcional: Habilitar API (¡Usar clave segura!)
#  api=yes
#  api-key=abrahackAPI123
```

## Configurar Zonas de Dominio

```bash
sudo pdnsutil create-zone abrahack ns1.abrahack
sudo pdnsutil add-record abrahack www A 10.100.100.10
```
---

## Iniciar PowerDNS y Verificar
```bash
sudo systemctl start pdns
sudo systemctl enable pdns
sudo systemctl status pdns
```

Verificación Inicial:
```bash
dig www.abrahack @127.0.0.1 -p 53053  #  Verificar localmente
```

## Configuración de DNSCrypt

Archivo: `/etc/dnscrypt-proxy/dnscrypt-proxy.toml`

```bash
listen_addresses = ['127.0.0.1:5353']
server_names = ['cloudflare']

ipv4_servers = true
require_dnssec = true
require_nolog = true
require_nofilter = true
```

Habilitar DNScrypt de nuevo

```bash
sudo systemctl start dnscrypt-proxy
sudo systemctl enable dnscrypt-proxy
sudo systemctl status dnscrypt-proxy
```

## Reglas IPTables


```bash
## Para wg0 → PowerDNS (y forward a DNSCrypt)
sudo iptables -t nat -A PREROUTING -i wg0 -p udp --dport 53 -j DNAT --to-destination 127.0.0.1:53053
sudo iptables -t nat -A PREROUTING -i wg1 -p udp --dport 53 -j REDIRECT --to-ports 5353
```
debes eliminar

``` bash
sudo iptables -t nat -I OUTPUT -d 10.100.100.0/24 -j RETURN
```

Guarda las reglas

```bash
sudo netfilter-persistent save
```

En el archivo de configuración del cliente WireGuard para wg0
DNS = 10.100.100.1
En el archivo de configuración del cliente WireGuard para wg1
DNS = 10.200.200.1

## Observaciones finales

- `wg0`: Red con resolución jerárquica y controlada.
- `wg1`: Red completamente anónima.
- Se pueden añadir más zonas en PowerDNS fácilmente.
- DNSCrypt funciona como filtro de privacidad externo confiable.
