# Configuración Completa de BIND9 + DNSCrypt + Zonas Públicas y Locales

Este documento guía la instalación y configuración de **BIND9** en una instancia EC2 de AWS como:
- 🔐 Resolutor DNS local para VPN wg0.
- 🌐 Servidor DNS autoritativo para dominios públicos (como abrahack.com).
- 🔁 Reenvío de consultas externas a DNSCrypt.
- 🔒 Integración con iptables, rangos de IP WireGuard y gestión segura de puertos.


---

## 1. Instalación y Preparación

```bash
sudo apt update 
sudo apt install bind9 bind9utils bind9-doc -y
```

## 2. Configura el archvio `named.conf.options`

```bash
sudo nano /etc/bind/named.conf.options
```

```bash
options {
    directory "/var/cache/bind";

    // Escuchar en puerto 53053 solo desde wg0
    listen-on port 53053 { 127.0.0.1; 10.100.100.1; };

    // Permitir consultas desde cualquier IP
    allow-query { any; };

    // Habilitar reenvío para dominios públicos
    recursion yes;
    forwarders {
        127.0.0.1 port 5353;
    };

    dnssec-validation auto;
    auth-nxdomain no;    # conform to RFC1035
    listen-on-v6 { none; };
};
```
## 3. Configurar zona local

- Zona Local: abrahack.wg0

Crea el directorio
```bash
sudo mkdir -p /etc/bind/zones
```

Configura el archivo
```bash
sudo nano /etc/bind/zones/db.abrahack.wg0
```

```bash
$TTL 604800
@   IN  SOA abrahack.wg0. root.abrahack.wg0. (
            1     ; Serial
        604800     ; Refresh
         86400     ; Retry
       2419200     ; Expire
        604800 )   ; Negative Cache TTL
;
@       IN  NS    ns1.abrahack.wg0.
ns1     IN  A     10.100.100.1
srv     IN  A     10.100.100.50
```

Añade en /etc/bind/named.conf.local:

```bash
zone "abrahack.wg0" {
    type master;
    file "/etc/bind/zones/db.abrahack.wg0";
};
```

## Edita reglas Iptables
eliminar regla previa
sudo iptables -t nat -I OUTPUT -d 10.100.100.0/24 -j RETURN

# Puerto TCP/UDP 53053 solo para wg0
sudo iptables -A INPUT -i wg0 -p udp --dport 53053 -j ACCEPT
sudo iptables -A INPUT -i wg0 -p tcp --dport 53053 -j ACCEPT

# Redirección DNS de wg0
sudo iptables -t nat -A PREROUTING -i wg0 -p udp --dport 53 -j DNAT --to-destination 10.100.100.1:53053

Verificaciones
sudo named-checkconf
sudo systemctl restart bind9
sudo systemctl enable named.service ## Bind9 es un enlace simbolico

Validación
Desde cliente wg0:
dig google.com @10.100.100.1 -p 53053
