# AWS(EC2) + DNSCRYPT + (VPN)

## Información general

### ¿Por qué usar DNScrypt?

- Resuelve tu tráfico DNS de forma cifrada, sin logs y sin censura, incluso dentro de tu VPN.
- El tráfico DNS es una de las formas más comunes de filtración de metadatos, incluso si ya usás Tor o VPN.
- Te permite verificar con precisión qué servicios están resolviendo qué dominios, y empezar a aplicar reglas.

¿Qué ventaja te da implementar primero DNSCrypt?

- Complementa y fortalece tu configuración actual de privacidad.
- Sirve como base técnica y conceptual para después montar:
- Tu propio servidor de DNS autoritativo (hosting de dominios).
- Tu servidor de correo cifrado y autenticado con SPF/DKIM/DMARC.
- Podés reutilizar el servidor como forwarder o resolutor seguro para todos tus flujos de n8n.
- Te brinda control absoluto de la resolución DNS para tus dispositivos y tus clientes (VPN).

🗂️ Camino Sugerido en orden
✅ DNSCrypt Proxy completo, funcionando en 10.100.100.1 y 10.200.200.1 y redirigido desde iptables.
✅ Pruebas con herramientas como dig, curl, tcpdump desde tu red local (PC y Android).
⚙️ Sistema de nombres propio (autoridad sobre tu propio dominio o TLD personalizado con BIND o PowerDNS).
📬 Servidor de correo en tu dominio, con:
- Autenticación segura.
- Alias privados.
- Cifrado oportuno (MTA-STS, DANE si es posible).
- Integración de esas cuentas y dominios en tus automatizaciones con n8n.

## 1. Instalación de dnscrypt-proxy

```bash
sudo apt update
sudo apt install -y dnscrypt-proxy
```

---

## 2. Configuración 

edita el archivo `/etc/dnscrypt-proxy/dnscrypt-proxy.toml`

```bash
server_names = ['cloudflare']
listen_addresses = ['10.200.200.1:5353']
ipv4_servers = true
require_dnssec = true
require_nolog = true
require_nofilter = true

[query_log]
file = '/var/log/dnscrypt-proxy/query.log'

[nx_log]
file = '/var/log/dnscrypt-proxy/nx.log'

[sources.'public-resolvers']
url = 'https://download.dnscrypt.info/resolvers-list/v2/public-resolvers.md'
cache_file = '/var/cache/dnscrypt-proxy/public-resolvers.md'
minisign_key = 'RWQf6LRCGA9i53mlYecO4IzT51TGPpvWucNSCh1CBM0QTaLn73Y7GFO3'
refresh_delay = 72
prefix = ''
```

---

## 3.Reglas iptables DNS

```bash
## Redirección de peticiones DNS en wg1 (UDP 53) a dnscrypt-proxy en :5353
sudo iptables -t nat -A PREROUTING -i wg1 -p udp --dport 53 -j REDIRECT --to-port 5353
```

Guarda las reglas

```bash
sudo netfilter-persistent save
```

---

# 4. Reiniciar y habilitar dnscrypt-proxy

```bash
sudo systemctl enable dnscrypt-proxy
sudo systemctl restart dnscrypt-proxy
```

## 5. Verificaciones

```bash
# Verificar escucha de DNSCrypt
sudo ss -nulp | grep 5353

# Resolver DNS desde cliente WireGuard (EC2)
dig check.torproject.org @10.200.200.1 -p 5353

# Comprobar conexión DNS desde Android conectado por wg1
curl https://check.torproject.org -x http://10.200.200.1:8443
```
